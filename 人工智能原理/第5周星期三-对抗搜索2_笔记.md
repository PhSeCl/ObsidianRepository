# 第5章 对抗搜索与博弈（二）：评估函数、Expectimax、MCTS

> [!info] 本章导览
> [[第4周星期五-对抗搜索1_笔记|Minimax+α-β]] 仍需搜到终局，对象棋（$35^{50}$）、围棋（b>300）力不从心。本节三步走：① **截断搜索 + 评估函数**应对"搜不到底"；② **Expectimax**应对"有随机性"；③ **蒙特卡洛树搜索（MCTS）**应对"分支因子巨大"。

## 截断搜索与评估函数

> [!important] 截断搜索（Depth-limited Search）
> 真实游戏搜不到终止状态 ⟹ 预设**深度限（horizon）**，用**评估函数**给非终止状态打分。
> - 不再保证最优决策，是**速度与质量的权衡**。
> - 例：象棋每步 100 秒、每秒 10K 结点 ⟹ 每步约 1M 结点；α-β 下 $35^{8/2}\approx1M$ ⟹ 可搜**深度 8**。

> [!note] 评估函数：加权线性特征和
> $$\text{EVAL}(s)=w_1 f_1(s)+w_2 f_2(s)+\dots+w_n f_n(s)$$
> 例（象棋）：$\text{Eval}=\text{material}+\text{mobility}+\text{king-safety}+\text{center-control}$，
> 其中 $\text{material}=9(Q-Q')+5(R-R')+3(B-B'+N-N')+1(P-P')$（皇后权重 9 等）。
> 也可用非线性函数（如神经网络，AlphaGo）。

> [!warning] 两个陷阱
> - **深度越深 → 通常决策越好**：更深搜索能中和评估函数的不准确。这是**计算复杂度（深度）vs 特征复杂度（评估函数）**的权衡。
> - **Thrashing（来回扭动）**：若评估函数只看"最终得分"，深度 2 时"现在吃豆"和"待会吃豆"看起来一样好 ⟹ 犹豫不决、原地扭动。**修正**：评估函数额外考虑"离豆子的距离"等中间特征。

## Expectimax：应对随机性

> [!important] 何时不知道动作结果？
> - **显式随机**：掷骰子。
> - **不可预测的对手**：吃豆人里随机行动的鬼。
> - **动作可能失败**：机器人轮子打滑。
>
> 此时状态值应反映**平均情况（expectimax）**，而非最坏情况（minimax）。

> [!note] Expectimax：增加"机会节点（Chance Node）"
> - MAX 节点同 Minimax 取 max。
> - **机会节点**像 MIN 节点，但取子结点的**期望（加权平均）**而非最小值。
> ```python
> def value(state):
>     if terminal: return utility(state)
>     if next is MAX: return max-value(state)
>     if next is EXP: return exp-value(state)
>
> def exp-value(state):
>     v = 0
>     for s' in successors(state):
>         p = probability(s')
>         v += p * value(s')      # 期望
>     return v
> ```

> [!example] 机会节点期望计算
> 子结点值 8、24、−12，概率 1/2、1/3、1/6：
> $$v=\tfrac12(8)+\tfrac13(24)+\tfrac16(-12)=4+8-2=10$$

> [!tip] Expectiminimax 与剪枝
> - **Expectiminimax**（如双陆棋）：环境是额外的"随机 agent"，在 min/max 之后行动；每个节点按类型取 max / min / 期望。
> - Expectimax **一般不能像 α-β 那样剪枝**，除非已知效用值的**取值范围**（有界时才有剪枝机会）。

![双陆棋（Backgammon）局面：黑棋顺时针走向 25、白棋逆时针走向 0；掷 6-5 时有四种合法走法——典型的"随机博弈"，每步前先掷骰子（机会节点）](./assets/第5周星期三_对抗搜索2_p37_1.jpg)

> [!note] 随机博弈中搜索的局限
> 双陆棋 2 骰子有 21 种结果、约 20 合法走法，4 层后 $20\times(21\times20)^3\approx1.2\times10^9$。
> - 深度增加时到达指定结点的概率骤降，**搜索作用减弱**。
> - 历史 AI：**TDGammon** 用 depth-2 搜索 + 好评估函数 + 强化学习，达世界冠军级——首个此类游戏 AI 世界冠军。

## 蒙特卡洛树搜索（MCTS）：应对巨大分支因子

> [!important] 为什么需要 MCTS
> α-β 假设固定深度 + 好评估函数；但**围棋 b>300**，且难以设计好评估函数。MCTS 结合两大思想：
> 1. **Rollout（试运行）评估**：从状态 s 用简单快速策略玩到终局多次，统计胜率 ≈ 真实赢的概率。
> 2. **选择性搜索（Selective search）**：把模拟资源集中到**有希望**的结点，不拘泥深度。

> [!note] 朴素版：纯 rollout 的问题
> - 对根的每个子结点跑 N 次 rollout，选胜率最高者。
> - 改进①：胜率太低的分支别再浪费资源。
> - 改进②（探索 vs 利用）：胜率相近但**模拟次数少**的结点估计**方差大**，应多分配模拟以更有信心。

> [!important] UCB1 公式：平衡"有希望"与"不确定"
> $$\text{UCB1}(n)=\underbrace{\frac{U(n)}{N(n)}}_{\text{利用：当前胜率}}+\;C\times\underbrace{\sqrt{\frac{\log N(\text{PARENT}(n))}{N(n)}}}_{\text{探索：次数越少越大}}$$
> - $N(n)$：从 n 出发的 rollout 次数；$U(n)$：累计效用（如赢的次数）。
> - 第一项偏好高胜率（利用），第二项偏好少访问（探索）。源自**多臂老虎机（bandit）**问题。

> [!important] MCTS（UCT）的四个阶段，重复至超时
> ![MCTS 四阶段：(a) Selection 用 UCB 递归选到叶节点；(b) Expansion+Simulation 加新子节点并 rollout（图中黑棋赢）；(c) Backpropagation 把胜负沿路径回传更新计数](./assets/第5周星期三_对抗搜索2_p56_1.jpg)
> 1. **选择（Selection）**：从根用 UCB 递归向下选到一个未完全扩展的叶节点 n。
> 2. **扩展（Expansion）**：给 n 加一个新子结点 c。
> 3. **模拟（Simulation/Rollout）**：从 c 跑一次 rollout 到终局。
> 4. **反向传播（Backpropagation）**：把结果沿 c→根路径更新各结点的胜/负计数。
> 最终：选择**访问次数 N 最高**的子结点对应的动作。

> [!tip] MCTS 的理论保证
> 结点"价值"$U(n)/N(n)$ 是子结点值的加权和；当 $N\to\infty$，绝大多数 rollout 集中在最优子结点，加权平均 → max/min。
> **定理**：$N\to\infty$ 时 UCT 选择的就是 Minimax 走法（但实际 N 永远到不了无穷）。AlphaGo 正是 MCTS + 神经网络。

## 本章小结

> [!summary] 关键要点
> 1. **截断搜索 + 评估函数**应对搜不到底；评估函数常用加权线性特征和；深度越深越能中和其不准。
> 2. **Expectimax** 用**机会节点取期望**应对随机性；一般不能剪枝（除非效用有界）。
> 3. **MCTS** 用 **rollout 统计胜率 + 选择性搜索**应对巨大分支因子。
> 4. **UCB1** 平衡"利用（高胜率）"与"探索（少访问）"。
> 5. MCTS 四步：选择→扩展→模拟→反向传播；$N\to\infty$ 时收敛到 Minimax。
