# 第5章 对抗搜索与博弈（一）：Minimax 与 α-β 剪枝

> [!info] 本章导览
> 前面是**单 Agent** 搜索；本章进入**多 Agent、目标冲突**的环境——**博弈**。先界定博弈的环境维度（确定/完全信息/两人/轮流/零和），引出 **Minimax**：在面对理性对手时倒推每个状态的最优值；再用 **Alpha-Beta 剪枝**把指数级搜索砍掉无用分支。

## 5.1 博弈的环境维度

> [!important] AI 中"博弈"的典型设定
>
> | 维度 | 取值 |
> |---|---|
> | 确定性 | **确定性**（vs 随机） |
> | 可观测 | **完全信息**（完全可观测） |
> | 玩家数 | **两人** |
> | 行动方式 | **轮流**（vs 同时） |
> | 效用关系 | **零和**：一方赢则另一方输，效用相反 |
>
> **零和游戏（Zero-Sum）**：纯竞争，一方**最大化**、另一方**最小化**同一个效用。
> 对比**一般博弈**：效用独立，可有合作、冷漠、竞争、联盟等。

> [!note] 博弈的形式化
> - **状态 s**、**Player(s)**（轮到谁）、**Actions(s)**（合法动作）、**Result(s,a)**（转移）、**Terminal-Test(s)**（终止）、**Utility(s,p)**（终局对玩家 p 的数值）。
> - 一个玩家的"解"是**策略（policy）** $S\to A$：每个状态推荐一个动作。
>
> 例（国际象棋）：状态=棋子位置+轮谁；效用=白胜+1/和0/负-1。

> [!example] 博弈 AI 发展水平
> Checkers（2007 完全解出）、Chess（1997 深蓝胜卡斯帕罗夫；2024 Stockfish 评分 3631）、Go（2016 AlphaGo 胜人类冠军）。

## 5.2 Minimax：面对理性对手的最优值

> [!important] 状态值 V(s) 的递归定义
> - **MAX 的轮次**（自己，最大化）：$V(s)=\max_{s'\in\text{succ}(s)}V(s')$
> - **MIN 的轮次**（对手，最小化）：$V(s)=\min_{s'\in\text{succ}(s)}V(s')$
> - **终止状态**：$V(s)=\text{Utility}(s)$（已知）。
>
> 即"面对理性敌人，从终止状态**倒推**回来的最佳可达效用"。

![Minimax 博弈树：蓝色三角为 MAX 层、红色倒三角为 MIN 层；叶子效用 3,12,8 / 2,4,6 / 14,5,2，MIN 层取各组最小得 3,2,2，MAX 层取最大得根值 3](./assets/第4周星期五_对抗搜索1_p27_1.jpg)

> [!example] 上图的倒推
> - MIN 层：min(3,12,8)=**3**，min(2,4,6)=**2**，min(14,5,2)=**2**。
> - MAX 层（根）：max(3,2,2)=**3**。最优首步走向值为 3 的分支。

### Minimax 算法（互相递归）

```python
def value(state):
    if terminal(state): return utility(state)
    if next_agent is MAX: return max-value(state)
    if next_agent is MIN: return min-value(state)

def max-value(state):           # 自己（最大化）
    v = -∞
    for s' in successors(state): v = max(v, value(s'))
    return v

def min-value(state):           # 对手（最小化）
    v = +∞
    for s' in successors(state): v = min(v, value(s'))
    return v
```

> [!note] 复杂度
> 设最大深度 m、分支因子 b：**时间 $O(b^m)$、空间 $O(bm)$**（DFS 式）。
> 国际象棋 $b\approx35,\ m\approx100$ ⟹ 状态数指数爆炸，**无法完整搜索** ⟹ 需要剪枝/启发式。

> [!warning] Minimax 假设对手"完美"
> Minimax 对**完美对手**最优。若对手并非完美（如随机鬼），用 Minimax 可能过于保守——后续 **Expectimax** 会按概率取期望，更适合应对随机对手。

## Alpha-Beta 剪枝

> [!important] 核心思想：不影响结果的枝就别看
> - **α**：到目前为止路径上 **MAX 已找到的最佳（最大）值**。
> - **β**：到目前为止路径上 **MIN 已找到的最佳（最小）值**。
> - **剪 MIN 结点**：算 MIN 结点 n 时，若其当前值已 **≤ α**，则上层 MAX 已有更好选择 m，理性 MAX 永不会走到 n ⟹ **停止检查 n 的其余后代**。
> - **剪 MAX 结点**：对称地，当前值 **≥ β** 即可剪。

![Alpha-Beta 剪枝同一棵树：根 α=3 后，中、右 MIN 结点一旦出现 β=2 ≤ α=3，其后续子结点（剪刀处）无需再看即可剪掉](./assets/第4周星期五_对抗搜索1_p38_1.jpg)

> [!example] 剪枝过程
> - 左 MIN 算得 3，根 MAX 的 **α=3**。
> - 中 MIN 第一个子是 2 ⟹ β=2 ≤ α=3 ⟹ 该 MIN 不可能超过 2 < 3，**剩余子结点全剪**。
> - 右 MIN 同理，遇到 2 即可剪。
> - 最终根值仍是 3，与完整 Minimax **结果完全一致**。

### Alpha-Beta 伪代码

```python
def max-value(state, α, β):
    v = -∞
    for s' in successors(state):
        v = max(v, value(s', α, β))
        if v ≥ β: return v          # 剪枝：MIN 不会让 MAX 拿到这么大
        α = max(α, v)
    return v

def min-value(state, α, β):
    v = +∞
    for s' in successors(state):
        v = min(v, value(s', α, β))
        if v ≤ α: return v          # 剪枝：MAX 不会走到这么小的 MIN
        β = min(β, v)
    return v
# 初始调用：max-value(root, -∞, +∞)
```

> [!tip] 剪枝效率取决于"看子结点的顺序"
> - **顺序越好，剪掉的枝越多**。理想顺序下时间复杂度可从 $O(b^m)$ 降到 $O(b^{m/2})$——相当于**分支因子开根号**，能搜索约**两倍深**。
> - α-β 剪枝**不改变最终的 Minimax 值**，只是少算了注定无用的分支。

## 本章小结

> [!summary] 关键要点
> 1. 博弈 = 多 Agent、目标冲突；典型设定：确定、完全信息、两人、轮流、**零和**。
> 2. **Minimax**：MAX 取 max、MIN 取 min，从终止状态倒推最优值；对完美对手最优。
> 3. Minimax 时间 $O(b^m)$ 指数级，象棋无法穷举。
> 4. **Alpha-Beta 剪枝**用 α/β 剪掉注定无用的枝，**结果不变**；好的子结点顺序下复杂度降到 $O(b^{m/2})$。
