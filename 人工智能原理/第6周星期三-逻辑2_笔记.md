# 第7章 逻辑 Agent（二）：命题逻辑推理

> [!info] 本章导览
> 承接[[第5周星期五-对抗搜索3+逻辑1_笔记|逻辑入门]]，本节解决"如何判断 KB ⊨ q"。先看如何用命题逻辑**建知识库**（吃豆人例），再讲三套推理引擎：**模型检验（真值表枚举）**、**前向链接（Horn 子句）**、**归结 + DPLL（基于 SAT）**，并理解 **CNF** 这一关键规范形式。

## 用命题逻辑建知识库：吃豆人例

> [!example] 吃豆人在已知地图中的逻辑求解
> 吃豆人知道地图，但只能感知上下左右有无墙。用命题词建模：
>
> | 类别 | 命题词 | 数量 |
> |---|---|---|
> | 墙的位置 | $\text{Wall}_{x,y}$ | N |
> | 感知 | $\text{Blocked\_W}_t$（t 时刻向西被挡） | 4T |
> | 行动 | $W_t,E_t,N_t,S_t$ | 4T |
> | 位置 | $\text{At}_{x,y,t}$ | NT |
> 共 $O(NT)$ 个变量 ⟹ $O(2^{NT})$ 个可能世界（N=200,T=400 时约 $10^{24000}$）。

> [!note] KB 该写哪些语句（约束掉"荒谬世界"）
> - **已知墙的位置**：$\text{Wall}_{0,0}\land\text{Wall}_{0,1}\land\dots$；以及无墙处 $\neg\text{Wall}_{1,1}\land\dots$
> - **一定在某处**：$\text{At}_{1,1,0}\lor\text{At}_{1,2,0}\lor\dots$（析取，至少一个为真）
> - **感知 ↔ 世界**：$\text{Blocked\_W}_0 \Leftrightarrow ((\text{At}_{1,1,0}\land\text{Wall}_{0,1})\lor\dots)$
> - **后继状态公理（successor-state axiom）**：状态变量怎么随时间变化
>   $$X_t \Leftrightarrow [X_{t-1}\land\neg(\text{某动作使它变假})]\lor[\neg X_{t-1}\land(\text{某动作使它变真})]$$
>
> ⚠️ 命题逻辑**表达力有限**：转移模型要 $O(NT)$ 条语句（约 20000 页！）——**一阶逻辑只需 O(1) 条**。实践中用代码自动生成这些语句。

## 推理引擎一：模型检验（Model Checking）

> [!important] TT-ENTAILS：真值表枚举
> 枚举所有模型（对全部命题词的真假赋值），验证"**在 KB 为真的每个模型里，α 也为真**"。
> ```text
> function TT-ENTAILS?(KB, α):           # 判断 KB ⊨ α
>     symbols ← KB 和 α 中的所有命题词
>     return TT-CHECK-ALL(KB, α, symbols, {})
>
> function TT-CHECK-ALL(KB, α, symbols, model):
>     if symbols 全部已赋值:
>         if PL-TRUE?(KB, model): return PL-TRUE?(α, model)
>         else: return true            # KB 为假的行直接放过（前件假蕴涵真）
>     else:
>         P ← symbols 第一个; rest ← 其余
>         return TT-CHECK-ALL(KB,α,rest, model∪{P=true})
>            and TT-CHECK-ALL(KB,α,rest, model∪{P=false})
> ```
> - 本质是命题词上的 **DFS 枚举**，**时间指数级 $O(2^n)$**。
> - 例（Wumpus）：128 行中只有 3 行 KB 为真，这 3 行里 $P_{1,2}$ 都为假 ⟹ **能推出 [1,2] 无坑**；但 $P_{2,2}$ 有真有假 ⟹ **推不出 [2,2] 是否有坑**。

## 推理引擎二：前向链接（Forward Chaining）

> [!important] 限定子句（Definite Clause）/ Horn 子句
> 前向链接要求 KB 只含**限定子句**：$(p_1\land\dots\land p_k)\rightarrow q$（前提是正文字合取，结论是单个正文字）。
> - ✅ 例：$(\text{Rain}\land\text{Snow})\rightarrow\text{Traffic}$；单个事实 $\text{Traffic}$（即 $\text{True}\rightarrow\text{Traffic}$）。
> - ❌ 非例：$\neg\text{Traffic}$；$(\text{Rain}\land\text{Snow})\rightarrow(\text{Traffic}\lor\text{Peaceful})$。
> - 等价形式：限定子句 = **恰含一个正文字的析取式** $(\neg X_1\lor\neg X_2\lor Y)$。

> [!note] 前向链接算法
> 反复对"前提已全部为真"的蕴含式应用 **Modus Ponens**，把结论加入已知事实，直到推出 q 或无法再推。
> ```text
> function PL-FC-ENTAILS?(KB, q):
>     count[c] ← 子句 c 前提中的符号数
>     inferred[s] ← false
>     queue ← KB 中已知为真的符号
>     while queue 非空:
>         p ← queue.pop()
>         if p == q: return true
>         if not inferred[p]:
>             inferred[p] = true
>             for 每个前提含 p 的子句 c:
>                 count[c] -= 1
>                 if count[c] == 0: queue.add(c.conclusion)
>     return false
> ```

![Horn 子句知识库的 AND-OR 图（Figure 7.16b）：A、B 为已知事实；带弧线的汇合表示 AND（如 A∧B→L、L∧M→P、B∧L→M），最终推出 P→Q。前向链接从底部 A、B 出发逐层点亮直到 Q](./assets/第6周星期三_逻辑2_p25_1.jpg)

> [!tip] 前向链接的性质
> 对限定子句 KB，前向链接**可靠且完备**。
> - 可靠性来自 Modus Ponens。
> - 完备性证明思路：到达**不动点**后，把推出的符号设 true、其余 false 构成模型 m；可证原 KB 每条子句在 m 中为真 ⟹ m 是 KB 的模型 ⟹ 任何被蕴含的原子语句在 m 中为真 ⟹ 都会被算法推出。

## 推理引擎三：归结、CNF 与 DPLL（基于 SAT）

> [!important] 归结（Resolution）与反证法
> 要证 $KB\models\alpha$，等价于证 **$KB\land\neg\alpha$ 不可满足（矛盾法）**：
> $$KB\models\alpha \iff KB\land\neg\alpha \text{ 在所有世界为假（unsatisfiable）}$$
> **归结规则**：对含**互补文字**的两个子句归结出新子句：
> $$\frac{\ell_1\lor\dots\lor\ell_k,\quad m}{\ell_1\lor\dots\lor\ell_{i-1}\lor\ell_{i+1}\lor\dots\lor\ell_k}\quad(\ell_i \text{ 与 } m \text{ 互补})$$
> 例：已知"食物在[1,1]或[3,1]"（$\ell_1\lor\ell_2$）且"食物不在[1,1]"（$\neg\ell_1$）⟹ 推出"食物在[3,1]"（$\ell_2$）。
> **命题逻辑归结是完备的**，但最坏情况指数级时间。

> [!note] 合取范式（CNF）
> 归结/SAT 求解器都在 **CNF** 上工作：
> - 整个语句 = 若干**子句的合取**（∧）；
> - 每个子句 = 文字的**析取**（∨）；
> - 每个文字 = 命题词或其否定。
>
> **任意语句 → CNF 的标准步骤**：① 消双条件 $\alpha\Leftrightarrow\beta\equiv(\alpha\Rightarrow\beta)\land(\beta\Rightarrow\alpha)$；② 消蕴涵 $\alpha\Rightarrow\beta\equiv\neg\alpha\lor\beta$；③ 内移否定（De Morgan）；④ ∨ 对 ∧ 分配。

> [!important] DPLL 算法（高效 SAT 求解）
> DPLL 本质是 TT-ENTAILS 的 DFS 枚举 + 三个加速启发式：
> 1. **及早终止（Early termination）**：所有子句已满足 → true；任一子句已假 → false。
> 2. **纯文字（Pure literal）**：在所有子句中只以同一符号位出现的符号，直接赋值满足它。如 A 在 $(A\lor B)\land(A\lor\neg C)$ 中纯正 ⟹ 设 A=true。
> 3. **单元子句（Unit clause）**：只剩一个文字未被赋假的子句，该文字必须取真。如 A=false 时 $(A\lor B)$ 退化为 $(B)$ ⟹ B=true。
> ```text
> function DPLL(clauses, symbols, model):
>     if 每个子句在 model 中为真: return true
>     if 某子句在 model 中为假: return false
>     P,val ← 找纯文字; if P: return DPLL(clauses, symbols-P, model∪{P=val})
>     P,val ← 找单元子句; if P: return DPLL(clauses, symbols-P, model∪{P=val})
>     P ← symbols 首个
>     return DPLL(...,{P=true}) or DPLL(...,{P=false})
> ```

## 本章小结

> [!summary] 关键要点
> 1. 命题逻辑建 KB：墙/位置/感知/后继状态公理；命题逻辑表达力有限（$O(NT)$ 条语句，一阶逻辑只需 O(1)）。
> 2. **模型检验 TT-ENTAILS**：枚举所有模型，指数级。
> 3. **前向链接**：要求 Horn/限定子句，反复用 Modus Ponens，可靠且完备。
> 4. **归结 + 反证**：证 $KB\land\neg\alpha$ 不可满足；命题逻辑归结完备。
> 5. **CNF** 是 SAT 的规范形式；**DPLL** = DFS + 及早终止 + 纯文字 + 单元子句。
